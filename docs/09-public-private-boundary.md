# 09 - Public / Private Boundary

This repository is a curated public technical showcase. It is not a mirror of
the private working monorepo.

## Included

- selected architecture docs;
- selected simplified source examples;
- public-safe diagrams;
- high-level process examples;
- validation and publishing checklists;
- technical review checklists.

## Excluded

- private monorepo history;
- full source data;
- generated runtime JSON;
- private media assets;
- backlog;
- prompts;
- skills;
- routers;
- internal methodology docs in full;
- deployment config;
- secrets;
- internal project details;
- local machine paths;
- unpublished Figma links;
- monetization notes;
- personal notes;
- raw scanner findings;
- task queues and session history.

## Publication Rule

If a file or excerpt is ambiguous, exclude it and list it as a manual review
item. A smaller public showcase is better than a public repository that leaks
private process or unreleased material.
